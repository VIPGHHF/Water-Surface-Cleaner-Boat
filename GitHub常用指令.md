# 树莓派 VS Code 终端 Git 同步速查表

## 第一阶段：保护案发现场（本地存档）
* `git status` : [雷达扫描] 查看树莓派上到底有哪些文件是新建的或被修改过的（红色的字）。
* `git add .` : [全部装箱] 把当前目录下所有变动的文件，全部放进暂存区。*(注意：`add` 后面必须有一个空格和英文句号 `.`)*
* `git commit -m "你的备注信息"` : [贴上封条] 给这次打包写上极其明确的备注（比如 `"新增了底层的 V4L2 摄像头测试脚本"`），在树莓派本地正式形成一个历史存档点。

## 第二阶段：天地大碰撞（拉取云端）
* `git pull origin main` : [降维合并] 把 GitHub 上的最新代码拉取到树莓派里。
  *(💡 极客注：如果你在电脑端也提交了新代码，这一步 Git 会自动把云端代码和树莓派本地代码“无缝缝合”。如果一切顺利，你会看到 `Fast-forward` 或 `Merge made` 的字样。如果报错提示分支名不对，可能你的主分支叫 `master`，那就换成 `git pull origin master`)*

## 第三阶段：火力全开（推送云端）
* `git push origin main` : [终极发射] 把树莓派上写好的代码，加上刚刚融合的云端代码，作为一个完整的“究极体”，一起推送到 GitHub 仓库！
