\# 🚤 Water-Surface-Cleaner-Boat (智能水面垃圾收集船)

> 基于 \*\*树莓派 (YOLO 视觉) + STM32 (双发差速控制)\*\* 的全自动水面垃圾清理机器人。



本项目由“大脑”与“小脑”分离的分布式架构组成：树莓派运行 Ubuntu 系统负责水面垃圾的视觉识别与航行决策，STM32F407 负责底层硬件的精确 PWM 控制与双体船的差速运动控制。



\---



\## 🛠️ 硬件架构 (BOM 清单)



为了在极小空间内实现高机动性与安全性，本项目采用了以下核心硬件选型：



\- \*\*基础载具\*\*：50x30cm 双体打窝船壳（天然吞噬通道，抗侧倾能力强）

\- \*\*核心控制大脑\*\*：Raspberry Pi 3B+ (运行 Ubuntu + 视觉模型)

\- \*\*底层驱动小脑\*\*：STM32F407 核心板

\- \*\*动力推进系统\*\*：12V 微型水下无刷推进器 × 2（正反桨配置，抵消反扭矩）

\- \*\*电机调速器\*\*：40A 双向无刷电调 × 2（\*\*极其重要：无刹车，直接正反向\*\*）

\- \*\*能源中枢\*\*：3S 航模动力锂电池 (11.1V, 5200mAh, 25C)

\- \*\*稳压模块\*\*：大功率航模 UBEC (输入 3S，输出 5V/5A 纯净电流)

\## 🚨 协同开发“生死铁律” (全组必读)



为了避免项目在开发中途烧毁主板或丢失代码，所有协作成员必须遵守以下规范：



\### ⚡ 1. 硬件防烧警告（动力组必看）

1\. \*\*剪红线大法\*\*：两个 40A 电调的信号线（插 STM32 那头）都会向外输出 5V 电压。\*\*必须将排线中的「红线」挑出并用绝缘胶布包死悬空！\*\* 绝对禁止插入 STM32，否则将与 UBEC 的 5V 产生冲突，瞬间烧毁主板！

2\. \*\*电池防饿死\*\*：必须在 3S 电池的平衡头上插上“BB响”低压报警器。电压一旦低于 3.0V/节，立刻停机充电，否则电池永久报废。



\### 💻 2. 代码合并铁律（STM32 组必看）

使用 STM32CubeMX 生成代码时，\*\*所有手写的 C 语言代码，必须且只能写在 `/\* USER CODE BEGIN xxx \*/` 和 `/\* USER CODE END xxx \*/` 之间！\*\*

写在注释块外部的任何代码，都会在下一次重新 Generate Code 时被无情抹除！

\## 📡 串口通信协议 (波特率: 9600)



树莓派与 STM32 之间采用极简的单字符串口指令进行解耦通信：



| 树莓派下发指令 (Char) | 航行动作 | STM32 底层 PWM 执行逻辑 (TIM2) |

| :---: | :--- | :--- |

| `'W'` | \*\*全速前进\*\* | 左电机 1700us (正)，右电机 1700us (正) |

| `'S'` | \*\*紧急倒车\*\* | 左电机 1300us (反)，右电机 1300us (反) |

| `'A'` | \*\*原地左转\*\* | 左电机 1300us (反)，右电机 1700us (正) |

| `'D'` | \*\*原地右转\*\* | 左电机 1700us (正)，右电机 1300us (反) |

| `'Q'` | \*\*停机/解锁\*\* | 左电机 1500us (停)，右电机 1500us (停) |



\*(注：电调上电时，STM32 默认输出 1500us 并维持 3 秒，等待电调发出“滴——”声安全解锁。)\*
